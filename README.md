## 간략 소개
취업기간의 공백은 '휴식'이 아니라 '리팩토링'이었습니다. 모바일과 웹을 넘나드는 탄탄한 구조를 설계합니다.

지난 실무 경험 후, 저는 개발자로서 더 멀리 가기 위해 "내 코드는 확장 가능한가?"라는 질문을 던졌습니다. 그 답을 찾기 위해 지난 6개월간 저는 단순한 기능 구현을 넘어, 소프트웨어 아키텍처를 깊이 있게 파고드는 시간을 가졌습니다. 기존에 익숙했던 GetX 패턴에 안주하지 않고, Flutter 생태계의 표준인 Riverpod 2.0과 DDD(Clean Architecture)로 기술 스택을 마이그레이션 했습니다. Domain, Data, Presentation Layer를 철저히 분리하여 유닛 테스트가 가능한 구조를 설계하고, 유지보수가 용이한 코드를 작성하는 데 집요하게 매달렸습니다.

저의 이러한 엔지니어링 역량은 모바일 앱에만 국한되지 않습니다. 저는 React를 비롯한 웹 프로그래밍 환경에서도 즉시 전력으로 투입될 수 있는 유연함을 갖추고 있습니다. Flutter를 통해 깊이 있게 파고든 선언형 UI(Declarative UI), 상태 관리 패턴, 그리고 컴포넌트 설계 원칙은 웹 프론트엔드 개발의 본질과 맞닿아 있기 때문입니다. 상태 관리의 흐름과 비동기 데이터 처리, 그리고 견고한 아키텍처 설계라는 '개발의 본질'을 이해하고 있기에, 앱 서비스뿐만 아니라 백오피스(Admin)나 웹 서비스가 필요한 상황에서도 주도적으로 문제를 해결할 수 있습니다.

단순히 화면을 그리는 코더가 아닌, 서비스의 안정성을 고민하는 엔지니어가 되기 위해 노력했습니다. FVM을 통한 환경 분리, Mocktail을 활용한 테스트 코드 작성, 그리고 GitHub Actions를 이용한 CI/CD 배포 자동화까지 구축해 본 경험은 제가 앱과 웹 어디에서든 '운영 가능한 소프트웨어'를 만들 준비가 되었음을 증명합니다. 모바일과 웹, 플랫폼의 경계 없이 비즈니스의 문제를 가장 효율적인 기술로 해결하는 든든한 팀원이 되겠습니다.
